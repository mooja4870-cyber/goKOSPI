# 사용자 시나리오 및 예외 흐름

본 문서는 사용자가 `goKOSPI` 리밸런싱 알림 시스템을 실제로 사용하며 의사결정을 내리는 세부 여정과 예외 처리 흐름을 다룹니다.

---

## 1. 기본 시나리오 (Happy Path)

### 단계 1: 데이터 백그라운드 수집 및 모니터링
* 시스템은 장마감 직후(15:40) 및 매시 정각에 코스피 종합 지수 및 50대 우량주의 시세 데이터를 자동으로 갱신합니다.
* 수급 수집기는 외국인/기관 순매수 금액 누적치를 갱신합니다.

### 단계 2: 과열(리밸런싱 주의) 알림 수신
* SK하이닉스의 3일 누적 가중 수익률이 ATR 20일 변동성의 2.5배를 초과하고, 당일 외인/기관의 순매수가 약화(개인 추격 매수 중심)된 것이 스캐닝 엔진에 감지됩니다.
* **시스템**: 대표님의 디스코드 서버 `#kospi-signals` 채널에 **[과열 주의/분할 매도 제안]** 알림을 발송합니다.
* **사용자**: 스마트폰으로 푸시 알림을 받고, SK하이닉스 보유 물량 중 20%를 지정가 매도하여 현금을 확보합니다.

### 단계 3: 조정(리밸런싱 소화) 모니터링
* 이후 3일 동안 SK하이닉스는 고점 대비 -4.8% 하락하며 물량을 소화합니다.
* 시스템은 매일 종가 기준 최고점 대비 하락률을 실시간 계산합니다.

### 단계 4: 리밸런싱 종료(재진입 기회) 알림 수신
* SK하이닉스의 하락폭이 4%를 넘어 4.8%에 도달하고, 단기 이평선 지지선 부근에서 외국인의 매수 유입세가 포착됩니다.
* **시스템**: 디스코드 채널에 **[리밸런싱 종료/분할 재매수 제안]** 알림을 발송합니다.
* **사용자**: 확보했던 현금을 투입해 하락한 가격에 SK하이닉스를 재매수하여 주식 수량을 늘립니다.

## 2. 예외 및 대응 흐름 (Alternative / Exception Path)

### 예외 1: 휩소(Whipsaw) 발생 - 초강세장 돌파
* **상황**: 지수나 종목이 과열권에 진입했으나, 초대형 호재(예: 반도체 실적 역사적 갱신 등)로 인해 조정 없이 폭등을 이어가는 상황.
* **시스템 대응**:
  * 가격 변동성 스코어는 과열이나, **외국인/기관의 매수 대금이 역사적 평균 상위 5%** 이내로 유지되는 강한 수급 추세일 때는 알림을 일시적으로 **유예(Hold)**합니다.
  * 지연 후 수급 강도가 꺾이는 시점까지 알림 시점을 미뤄 섣부른 숏 포지션 진입 및 조기 익절을 방지합니다.

### 예외 2: 데이터 API 장애
* **상황**: 시세 수집 라이브러리(Yahoo Finance 등)의 쿼리 제한 또는 네트워킹 문제로 데이터 취득 실패.
* **시스템 대응**:
  * 3회 자동 재시도(Retry)하며, 재시도 간격은 5초 $\rightarrow$ 15초 $\rightarrow$ 60초로 지수 대피합니다.
  * 최종 실패 시, 디스코드 관리자 채널로 즉시 시스템 에러 알림(`[API Error] KOSPI 데이터 수집에 실패했습니다. 연결을 확인하세요.`)을 송출하여 신호 누락을 최소화합니다.
