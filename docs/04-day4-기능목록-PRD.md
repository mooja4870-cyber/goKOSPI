# DAY 4 — 기능 목록 & PRD 작성

> 출처: [16일간의 기적 · 바이브 코딩 로드맵](https://project.aibizbotu.com/) · 4단계 (DAY 4 / PLAN)

## 요약

구축할 어플리케이션의 핵심 기능을 도출하고, 이를 바탕으로 구체적인 제품 요구사항 정의서(PRD: Product Requirement Document)를 설계하여 실제 개발의 설계 청사진을 완성합니다.

## 목표

- 시스템에 필요한 핵심 모듈 및 기능 목록(Feature List)을 정리한다.
- 데이터 모델, 알고리즘 공식, 연동 API 스펙을 포함한 PRD를 작성한다.

## 체크리스트

- [x] 핵심 기능 목록(시세 수집, 연산, 수급 분석, 알림)을 도출한다.
- [x] 각 기능별 상세 구현 명세와 데이터 흐름을 정의한다.
- [x] 기술 스택과 API 인터페이스 스펙을 확정하여 PRD로 정리한다.
